# 主页任务条幅左滑设置与删除功能修改说明

## 需求

主页任务条幅支持**左滑**，左滑后露出"设置"（蓝色）和"删除"（红色）两个按钮：
- 点击"设置"按钮：进入该任务的编辑设置页面（TaskEditPage）
- 点击"删除"按钮：从数据库和界面中删除该任务

## 修改概述

共涉及 **3 个文件**：

| 文件 | 改动类型 |
|------|---------|
| `entry/src/main/ets/view/HomeComponent.ets` | 功能新增 + Bug修复 |
| `entry/src/main/ets/view/home/TaskCardComponent.ets` | 功能调整 + Bug修复 |
| `entry/src/main/ets/ai/service/AiService.ets` | 原有编译错误修复 |

#### d) 新增 deleteTask 方法（第 127~145 行）

```typescript
deleteTask(item: TaskInfo) {
  let dateStr = item.date || dateToStr(new Date());
  let deleteInfo = new TaskInfo(item.id, dateStr, item.taskID, item.targetValue,
    item.isAlarm, item.startTime, item.endTime, item.frequency, item.isDone,
    item.finValue, false);
  TaskInfoTableApi.deleteDataByID(deleteInfo, (res: number) => {
    Logger.info('deleteTask', 'Delete result: ' + res);
    this.homeStore.dateArr = this.homeStore.dateArr.map((weekItem) => {
      if (weekItem.dateStr === dateStr) {
        weekItem.taskList = weekItem.taskList.filter((taskItem: TaskInfo) => taskItem.taskID !== item.taskID);
        if (this.homeStore.checkCurrentDay()) {
          this.homeStore.selectedDayInfo = weekItem;
        }
        return weekItem;
      }
      return weekItem;
    }).slice(0);
  });
}
```

**说明：**
1. 构造一个 `TaskInfo` 对象（`isOpen = false`）传给 `TaskInfoTableApi.deleteDataByID`，按 `date + taskID` 从数据库删除记录
2. 删除成功后，从 `homeStore.dateArr` 中对应日期的 `taskList` 中过滤掉该任务
3. 如果是当前日期，同步更新 `selectedDayInfo` 以刷新界面
4. `.slice(0)` 触发 ArkUI 的状态刷新机制，确保界面更新

- 移除了 `.margin({ bottom: 12 })`，改由 `List({ space: 8 })` 的 `space` 统一控制间距

### 3. `entry/src/main/ets/ai/service/AiService.ets`（原有编译错误修复）

此文件修改不属于本次功能需求，但因其原有代码存在编译错误导致整个项目无法构建，需一并修复。

#### a) 定义明确的类替代对象字面量

```typescript
class HttpHeader {
  contentType: string = '';
  authorization: string = '';
}

class HttpRequestConfig {
  method: http.RequestMethod = http.RequestMethod.POST;
  header: HttpHeader = new HttpHeader();
  extraData: string = '';
  connectTimeout: number = 30000;
  readTimeout: number = 60000;
}
```

**原因：** ArkTS 要求对象字面量必须对应显式声明的类或接口，原代码直接在 `httpRequest.request()` 参数中使用匿名对象字面量导致类型推断失败。

#### b) `body` 改为 `extraData`

```diff
- body: requestBody,
+ extraData: requestBody,
```

**原因：** `@ohos.net.http` 的 `HttpRequestOptions` 接口中请求体字段名为 `extraData`，不存在 `body` 属性。

#### c) 使用类实例替代对象字面量传参

```diff
- const response = await httpRequest.request(this.apiUrl, {
-   method: http.RequestMethod.POST,
-   header: {
-     'Content-Type': 'application/json',
-     'Authorization': `Bearer ${this.apiKey}`
-   },
-   body: requestBody,
-   connectTimeout: 30000,
-   readTimeout: 60000
- });
+ const config = new HttpRequestConfig();
+ config.method = http.RequestMethod.POST;
+ config.header = new HttpHeader();
+ config.header.contentType = 'application/json';
+ config.header.authorization = 'Bearer ' + this.apiKey;
+ config.extraData = requestBody;
+ config.connectTimeout = 30000;
+ config.readTimeout = 60000;
+ const response = await httpRequest.request(this.apiUrl, config);
```

**原因：**
- ArkTS 不允许在对象字面量中使用连字符属性名（如 `'Content-Type'`），改为使用驼峰命名的类属性
- 使用类实例替代匿名对象字面量，满足 ArkTS 类型约束

#### d) 全部模板字符串改为字符串拼接

```diff
- Logger.info(TAG, `configure: apiUrl=${apiUrl}, model=${model ? model : this.model}`);
+ Logger.info(TAG, 'configure: apiUrl=' + apiUrl + ', model=' + (model ? model : this.model));
```

**原因：** ArkTS 禁止使用模板字符串，全文件统一改为字符串拼接。
