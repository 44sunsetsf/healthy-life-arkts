# 每日刷牙 & 每日微笑 — 目标设置与多次提醒修改说明

## 修改概述

共涉及 **8 个文件**，实现四个修复/功能：
1. 每日刷牙和每日微笑的目标设置从**不可设置**改为**可设置 1~3 次**
2. 提醒次数与目标次数联动：**目标设几次，提醒就设几个时间点**
3. **修复提醒不响的 bug**（`isChanged` 永远为 `false` 导致设置从未保存）
4. **修复任务进度计算错误**（`targetTaskNum` 只计入已开启任务导致过早 100%）

---

### 1. `entry/src/main/ets/common/constants/CommonConstants.ets`

**新增常量：**

```diff
+ static readonly TIMES_MAX_RANGE: number = 3;
```

### 2. `entry/src/main/ets/model/TaskInitList.ets`

**改动：** 每日微笑和每日刷牙的 `step` 从 `0` 改为 `1`

```diff
  // 每日微笑 (taskID: 4)

  // 每日刷牙 (taskID: 5)
```

```typescript
  let existingTimes = this.settingParams?.startTime.split(',').map(t => t.trim());
```

## 修改后的行为

| 场景 | 目标 | 提醒时间 | 提醒次数 |
|------|------|---------|---------|
| 默认 | 1 次 | 08:00 | 1 个系统闹钟通知 |
| 用户改为 2 次 | 2 次 | 08:00, 20:00 | 2 个系统闹钟通知 |
| 用户改为 3 次 | 3 次 | 08:00, 12:30, 20:00 | 3 个系统闹钟通知 |

- 目标设置仅在任务**开启**时可编辑
- 提醒时间选择器数量 = 目标次数（1~3 个）
- 打卡逻辑：每次 +1，累加到目标值后完成

---

## 数据格式说明

`startTime` 字段从单个时间变为逗号分隔的多个时间：

| 旧格式 | 新格式 |
|--------|--------|
| `"08:00"` | `"08:00"`（1次时兼容） |
| — | `"08:00,20:00"`（2次） |
| — | `"08:00,12:30,20:00"`（3次） |

向下兼容：旧数据中 `startTime: "08:00"` 拆分后得到 `["08:00"]`，自动适配。

## Bug 修复

### 7. `entry/src/main/ets/view/task/TaskDetailComponent.ets` — 修复提醒不保存

**问题根因：** `@Watch('onParamsChanged')` 在子组件通过 `@Consume` 修改嵌套属性时**不会触发**（对象引用未变，只改了内部属性），导致 `isChanged` 永远为 `false`，`addTask()` 从未被调用，**设置从未保存到数据库，提醒从未发布**。

**修复：** 移除 `isChanged` 检查和 `@Watch`，点"完成"**始终保存**：

```diff
- @Provide @Watch('onParamsChanged') settingParams: ITaskItem = this.parseRouterParams();
+ @Provide settingParams: ITaskItem = this.parseRouterParams();

- private isChanged: boolean = false;

- onParamsChanged() {
-   this.isChanged = true;
- }

  finishTaskEdit() {
-   if (this.isChanged) {
      // ... addTask() 始终执行
-   }
-   router.back({ url: 'pages/MainPage' });
  }
```

### 8. `entry/src/main/ets/viewmodel/HomeViewModel.ets` — 修复任务进度计算

**问题根因：** `targetTaskNum = taskList.length`，而 `taskList` 只包含**已开启**的任务。如果用户只开启了 2 个任务，完成 2 个 → 2/2 = 100%，而不是 2/6 = 33%。

**修复：** `targetTaskNum` 始终设为 `Const.TASK_NUM`（6 = 总任务数）：

```diff
- dayInfo.targetTaskNum = taskList.length;
+ dayInfo.targetTaskNum = Const.TASK_NUM;

  // updateDayInfo() 中同样修改
- dayInfo.targetTaskNum = this.selectedDayInfo.taskList.length;
+ dayInfo.targetTaskNum = Const.TASK_NUM;

  // 关闭已完成任务时不再减少 finTaskNum（已完成的任务应持续计入进度）
- if (taskList[taskIndex]?.isDone) {
-   dayInfo.finTaskNum -= 1;
- }
```
